Authentication Concepts Demo

This application demonstrates different authentication concepts in a Flask app.

Stateful Authentication

Demonstration of server-side sessions stored in Redis.

  • Session data stored on server
  • Only session ID stored in client cookie
  • Includes Multi-Factor Authentication
  • Server maintains state
Try Session Login

Stateless Authentication

Demonstration of JWT token-based authentication.

  • All session data stored in token
  • Token stored in browser's localStorage
  • No MFA (simplified for demo)
  • Server is stateless
Try Token Login

HTTP Authentication Methods

Additionally, this demo includes examples of HTTP authentication methods:

Basic Authentication

Simple username/password authentication using HTTP headers.

Try Basic Auth
Digest Authentication

More secure authentication that doesn't send passwords in clear text.

Try Digest Auth

This is an educational demo. Do not use in production without proper security review.
